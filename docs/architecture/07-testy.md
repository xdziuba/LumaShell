# Testy

## Testy jednostkowe

* profile,
* konfiguracja,
* system motywów,
* walidacja manifestów,
* uprawnienia wtyczek,
* mapowanie skrótów,
* zarządzanie sesjami.

## Testy integracyjne

* PowerShell przez PTY,
* WSL,
* SSH,
* port COM,
* SFTP,
* instalacja wtyczki,
* awaria wtyczki,
* przywracanie sesji,
* aktualizacja aplikacji.

## Testy E2E

Do testów można wykorzystać **Playwright**.

Scenariusze:

* uruchomienie terminala,
* otwarcie nowej zakładki,
* wykonanie komendy,
* podział terminala,
* połączenie SSH,
* otwarcie portu COM,
* zmiana motywu,
* instalacja wtyczki,
* ponowne uruchomienie aplikacji.

## Testy wydajnościowe

Należy mierzyć:

* czas uruchamiania,
* wykorzystanie pamięci,
* obciążenie CPU,
* płynność przewijania,
* wydajność przy wielu kartach,
* przepustowość danych,
* opóźnienia IPC,
* wpływ wtyczek na działanie programu.

Cele liczbowe: [05 — Wydajność](05-wydajnosc.md).
